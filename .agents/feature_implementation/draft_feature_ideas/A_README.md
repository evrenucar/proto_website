# draft_feature_ideas/

Loose pool of feature ideas. Each file is a short sketch, not a full spec.

When an idea graduates to in-flight work, copy it into `feature_implementation/<feature_name>.md` and expand it into the full spec layout described in `feature_implementation/README.md`. Then delete or strike through the draft here.

## File format

```
# <Feature Idea Title>

## Problem / Why
1 to 3 sentences.

## Sketch
2 to 5 bullets describing the approach.

## Notes
Optional. References, similar tools, code areas to touch, or open questions.
```

## Conventions

- One idea per file.
- File name is kebab-case of the idea title, e.g. `inline-canvas-crop-presets.md`.
- Keep prose tight. This folder is a brainstorm pool, not a planning document.
- No em dashes. Period, comma, or a connector word instead.
