# AGENTS.md

## Session Workflow

- Read [current_scratch_pad.md](./current_scratch_pad.md) at the start of every work session.
- Read [.agents/agents.md](./.agents/agents.md) for project context and routing.
- Update `current_scratch_pad.md` at the start of the session with the current scope.
- Update `current_scratch_pad.md` again at the end of the session with what changed, what remains, and the next step.
- Treat `current_scratch_pad.md` as a shared working surface. The user may write there directly.

## Scope Rule

- Keep `current_scratch_pad.md` limited to what is being worked on now.
- Do not turn it into a long-term spec, changelog, or permanent knowledge base.

## Default Startup Context

A new agent should read only these three files by default:

1. This file (`AGENTS.md`)
2. `current_scratch_pad.md`
3. `.agents/agents.md`

Do not read archive, handoff, or backlog files unless the task specifically requires history.
