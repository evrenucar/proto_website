# content/entities

## Purpose
Holds instance and index data for site entities — the runtime list the site reads to resolve entity slugs to titles, types, and surface references.

## Read when
Looking up how entity data flows from source JSON into the site (build, preview, board rendering). Investigating which entities exist and what surfaces they appear on.

## Skip when
Defining or modifying what an entity TYPE is (go to `src/entities/` instead). Doing board layout, CSS, or unrelated feature work.

## Canonical for
The authoritative instance index (`index.json`) that maps entity slugs to their type, title, status, tags, and cross-surface references. This is the read side; `src/entities/` is the definition side.

## Key files
- `index.json` — generated index of all active entities; each entry carries `slug`, `type`, `title`, `status`, `tags`, `references`, and `sourcePath` back to `src/entities/<slug>.json`

## Conventions
- Do not hand-edit `index.json` if it is generated by the build; regenerate it via the build script instead.
- This directory holds index/instance data only — no type schemas, no renderers.
- `src/entities/<slug>.json` is the source of truth for each entity fields. `content/entities/index.json` is the derived read index.
- Do not confuse this directory with `src/entities/` (type definitions) — they serve opposite roles.
